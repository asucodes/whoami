WebRTC Watchparty with TURN for Surviving Restrictive Firewalls

2025 · ~5 min read
full repo (react + node + webrtc + socket.io) on github

streamit began as a small experiment in real-time shared viewing. one person would share their screen or camera while others watched with minimal delay and chatted in real time, complete with gifs and floating reactions. the stack was a react frontend, a node server using socket.io for signaling, and native webrtc through simple-peer for the actual media streams.

on a clean local network or a normal residential connection the peer-to-peer path performed very well. sub-second latency, smooth 1080p, and multiple viewers without the host machine struggling.

the first time we tried it across a campus or office network with a restrictive firewall the whole thing fell apart. webrtc prefers direct udp, and when the only thing the network reliably allows is tcp 443 that looks like ordinary tls traffic, the peer connection either never forms or falls back to something painfully slow.

the change that made it survive was adding a turn server (coturn) and including it in the webrtc ice server configuration as a relay of last resort. when the direct path was blocked, the clients would fall back to the turn candidate, the media would travel over the relayed connection, and the stream remained usable.

keeping the native webrtc path (instead of forcing everything through a custom layer) mattered because the browser itself gets to evaluate and select the best candidate pair. once the turn entry was present in the ice servers, the fallback was automatic and reliable rather than a complete failure.

the deployed instance is still up at watch.1petaflop.tech. the full source for both the react client and the node signaling server lives at https://github.com/asucodes/streamit. the short version is simple: use peer-to-peer when the network allows it, fall back to turn when the network is hostile, and keep the signaling layer straightforward.

that combination was enough to let the application work on the kinds of networks real people actually use.
